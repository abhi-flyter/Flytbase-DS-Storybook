# Phase 5 Storybook MCP Readiness

Phase 5 connects the F Design System Storybook to AI agents through Storybook MCP so agents can discover real components, read documented props, inspect examples, and avoid hallucinated UI APIs.

## MCP Server

- Local Storybook command: `npm run storybook`
- Local Storybook URL: `http://localhost:6006`
- Local MCP endpoint: `http://localhost:6006/mcp`
- Recommended MCP server name for agents: `fb-design-system-sb-mcp`

## Implemented

- `@storybook/addon-mcp` is installed.
- `@storybook/addon-vitest` is installed for browser-based Storybook tests.
- `.storybook/main.ts` registers `@storybook/addon-mcp`.
- `.storybook/main.ts` registers `@storybook/addon-vitest`.
- `.storybook/main.ts` enables `experimentalComponentsManifest` and `experimentalCodeExamples`.
- `storybook-static/manifests/components.json` is generated by `npm run build-storybook`.
- `vitest.config.ts` configures the Storybook Vitest project with Playwright Chromium.
- `npm run test-storybook` runs `vitest --project=storybook`.
- `AGENTS.md` contains the project-level Storybook MCP usage rules for agent-agnostic tools.
- `CLAUDE.md` contains the same rules for Claude-based workflows.
- `.mcp.json` declares the project-level `fb-design-system-sb-mcp` HTTP server at `http://localhost:6006/mcp`.
- `npm run verify:phase5` checks the Phase 5 file/config/static-manifest gates.
- `npm run verify:phase5:live` checks MCP initialization, tool discovery, documentation lookup, and component prop retrieval.
- With `STORYBOOK_MCP_URL=http://127.0.0.1:6208/mcp`, `npm run verify:phase5:live` also verifies the running Storybook MCP development/testing toolset and runs a focused story test through `run-story-tests`.
- `npm run test-storybook` passes locally with 32 test files and 65 tests.

## Agent Registration

With Storybook running, connect your local agent to the MCP endpoint:

Agents that support project-level `.mcp.json` can use the checked-in server entry directly:

```json
{
  "mcpServers": {
    "fb-design-system-sb-mcp": {
      "type": "http",
      "url": "http://localhost:6006/mcp"
    }
  }
}
```

For agents that prefer CLI registration, run:

```bash
npx mcp-add --type http --url "http://localhost:6006/mcp" --scope project
```

When prompted for a name, use:

```text
fb-design-system-sb-mcp
```

Claude Code users can alternatively run:

```bash
claude mcp add fb-design-system-sb-mcp --transport http http://localhost:6006/mcp --scope project
```

## Required Agent Behavior

Agents must:

- Never hallucinate component properties.
- Query `list-all-documentation` before choosing components.
- Query `get-documentation` before using a component prop.
- Query `get-documentation-for-story` when story-level evidence is needed.
- Use `get-storybook-story-instructions` before adding or updating stories when the development toolset is available.
- Run `run-story-tests` when connected through an agent that exposes the MCP testing toolset, or run `npm run test-storybook` locally.
- Ask the user before using undocumented props.

## Manual Confirmation

After registering the MCP server in your preferred agent, ask:

```text
List all documented components from our Storybook MCP server.
```

The agent should call `list-all-documentation` through `fb-design-system-sb-mcp` and return the F Design System component library rather than guessing from source files.
